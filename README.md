##  Описание проекта

Консольное приложение на C# (.NET), реализующее простой класс `BankAccount` для демонстрации работы с банковским счётом.  
Проект содержит методы для пополнения (`Credit`) и снятия (`Debit`) средств, а также точку входа `Main` для тестового запуска.

В ходе работы выполнялось:

- документирование кода с помощью XML-комментариев;
- создание модульных тестов (MSTest);
- тестирование методом "белого ящика";
- рефакторинг кода на основе результатов тестирования.

---

##  Основные тестируемые методы

| Метод       | Описание                          |
|-------------|-----------------------------------|
| `Debit()`   | Снятие денежных средств со счёта |
| `Credit()`  | Зачисление денежных средств      |

##  Установка и запуск

### Требования

- .NET 8.0 SDK
- Visual Studio 2022 (или любая другая IDE с поддержкой C#)
- ОС: Windows 10 / 11

### Клонирование репозитория

```bash
git clone https://github.com/Xesus777/Bank.git
cd Bank
```

### Сборка и запуск приложения

```bash
dotnet restore
dotnet build
dotnet run --project Bank/Bank.csproj
```

### Запуск тестов

```bash
dotnet test BankTests/BankTests.csproj
```

Или через **Обозреватель тестов** в Visual Studio.

##  Вывод о проведённом тестировании

В ходе выполнения практической работы были созданы модульные тесты для методов `Debit` и `Credit` класса `BankAccount`.

###  Тесты для метода `Debit`

1. **Debit_WithValidAmount_UpdatesBalance**  
   - *Результат:* Успешно  
   - *Причина:* После исправления ошибки (замена `+=` на `-=`) метод корректно уменьшает баланс.

2. **Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange**  
   - *Результат:* Успешно  
   - *Причина:* Метод выбрасывает исключение при отрицательной сумме.

3. **Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange**  
   - *Результат:* Успешно  
   - *Причина:* Метод выбрасывает исключение при попытке снять больше, чем есть на счёте.

4. **Debit_WhenAmountIsMoreThanBalance_ShouldThrowArgumentOutOfRange_WithCorrectMessage**  
   - *Результат:* Успешно  
   - *Причина:* После рефакторинга исключение содержит корректное сообщение.

5. **Debit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange_WithCorrectMessage**  
   - *Результат:* Успешно  
   - *Причина:* Исключение содержит сообщение о недопустимой отрицательной сумме.

---

###  Тесты для метода `Credit`

1. **Credit_WithValidAmount_UpdatesBalance**  
   - *Результат:* Успешно  
   - *Причина:* Метод корректно увеличивает баланс.

2. **Credit_WhenAmountIsLessThanZero_ShouldThrowArgumentOutOfRange**  
   - *Результат:* Успешно  
   - *Причина:* Метод выбрасывает исключение при отрицательной сумме.

3. **Credit_WithZeroAmount_UpdatesBalance**  
   - *Результат:* Успешно  
   - *Причина:* Зачисление нуля не меняет баланс (метод работает корректно).

4. **Credit_WithLargeAmount_UpdatesBalance**  
   - *Результат:* Успешно  
   - *Причина:* Метод корректно обрабатывает большие суммы.

---

##  Причины успешного / неуспешного выполнения тестов

| Тест | Статус | Причина |
|------|--------|---------|
| `Debit_WithValidAmount_UpdatesBalance` | ❌ (первый запуск) | Ошибка в логике: использовалось сложение вместо вычитания |
| `Debit_WithValidAmount_UpdatesBalance` | ✅ (после исправления) | Логика исправлена: `m_balance -= amount` |
| Все остальные тесты | ✅ | Код методов `Debit` и `Credit` написан корректно, исключения выбрасываются в нужных местах |
